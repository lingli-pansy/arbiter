用户正在构建一个私人交易辅助系统。

当前已经确定的底层选型：
- NautilusTrader 用于回测引擎。
- IB 用于券商执行。
- OpenClaw 用作真实基金经理 agent。
- Cursor 作为外包开发团队，主要根据本地 ticket 与文档工作。

当前系统建设目标：
- 不先追求万能 agent。
- 先让 OpenClaw 通过真实任务去使用系统。
- 任务受阻时，写 ticket 给 Cursor。
- Cursor 实现后，再由 OpenClaw 回头验收。

当前第一阶段优先级：
1. 跑通市场数据能力。
2. 跑通新闻数据能力。
3. 跑通回测能力。
4. 之后再做模拟调仓和更薄的交易辅助。

工作偏好：
- 一切尽量通过本地文档与仓库状态交互。
- 希望减少来回复制粘贴。
- 希望每个需求落地后都能立刻开始做具体事情，而不是继续抽象讨论。

## Gap Analysis

### 🔴 High Severity: 时间戳格式

**问题:** 
- Current: `"2024-01-01T00:00:00Z"` (ISO 8601 string)
- NT: `1704067200000000000` (nanoseconds since epoch)

**影响:**
- 无法直接将 current tool 输出喂给 NT 回测引擎
- 需要时间戳转换层

**建议:**
- 添加 `timestamp_to_nt_ts()` 转换函数
- 或在 `get_market_bars_batch` 中添加 `output_format: "nt_compatible"` 选项

### 🟡 Medium Severity: 类型系统

**问题:**
- Current: 原生 Python 类型 (float, int, str)
- NT: 领域类型 (Price, Quantity, BarType)

**影响:**
- 类型不匹配可能导致 NT 解析失败
- 需要类型转换

**建议:**
- 添加类型转换适配器
- 明确价格精度处理（decimal vs float）

### 🟡 Medium Severity: Bar 类型标识

**问题:**
- Current: 无 bar_type，symbol/timeframe 在 metadata 中
- NT: bar_type 是 Bar 的一部分，格式为 "SYMBOL.VENUE-TIMEFRAME-AGG"

**影响:**
- NT 需要 bar_type 来路由数据

**建议:**
- 构建 bar_type 生成逻辑：`{symbol}.{venue}-{timeframe}-LAST`

### 🟢 Low Severity: 修订标记

**问题:**
- Current: 无 revision 概念
- NT: is_revision 标记 bar 是否为修订版

**影响:**
- 仅影响实时数据校正场景
- 回测场景可忽略

## Alignment Recommendations

### Option 1: Adapter Pattern (Recommended)

创建 `convert_to_nt_bars()` 工具，将 current output 转为 NT Bar 格式：

```python
def convert_to_nt_bars(market_data: dict) -> list[Bar]:
    """Convert get_market_bars_batch output to NautilusTrader Bar objects"""
    for symbol, bars in market_data.items():
        for bar in bars:
            yield Bar(
                bar_type=f"{symbol}.NASDAQ-1-MINUTE-LAST",  # 需要 venue 映射
                open=str(bar["open"]),
                high=str(bar["high"]),
                low=str(bar["low"]),
                close=str(bar["close"]),
                volume=str(bar["volume"]),
                ts_event=iso_to_ns(bar["timestamp"]),
                ts_init=iso_to_ns(bar["timestamp"]),
            )
```

### Option 2: Extend Current Tool

在 `get_market_bars_batch` 中添加参数：

```yaml
output_format:
  type: string
  enum: ["native", "nt_compatible"]
  default: "native"
```

### Option 3: Separate NT Data Path

为 NT 独立实现 `get_market_bars_nt()`，使用 NT 原生数据适配器。

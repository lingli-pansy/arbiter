# TICKET_20250314_001_FOLLOWUP_004

## Source Task
VALIDATION_20250314_001_ARCH_FIX - Yahoo Finance 限流

## Blocking Issue
使用 Yahoo 数据源时返回:
```
Too Many Requests. Rate limited. Try after a while.
```

## Root Cause
Yahoo Finance 对未认证请求有严格频率限制:
- 约 20 只股票/分钟 后开始限流
- 候选池 20 只股票导致立即触发限制

## Proposed Solutions

### 方案 A: 添加延迟 (推荐短期)
在股票循环中添加 1-2 秒延迟:
```python
for sym in syms:
    try:
        t = yf.Ticker(sym)
        ...
    finally:
        time.sleep(1.5)  # 延迟 1.5 秒
```

### 方案 B: 使用 yfinance 批量接口
使用 `yf.download` 一次性获取多只股票:
```python
data = yf.download(syms, start=start, end=end, group_by='ticker')
```

### 方案 C: Session 复用
复用 HTTP session 减少连接开销:
```python
import requests
session = requests.Session()
ticker = yf.Ticker(sym, session=session)
```

### 方案 D: 本地缓存
缓存市值数据，减少重复请求:
```python
# 缓存格式: {date: {symbol: market_cap}}
CACHE_TTL = 86400  # 24小时
```

## Acceptance Criteria
- [ ] 能成功获取 20 只候选股票的市值而不触发限流
- [ ] 延迟可接受 (< 30 秒完成全部请求)
- [ ] 有基本错误处理（限流时自动重试或返回部分结果）
